# HTML

# H5 离线存储 manifest

<html manifest="cache.manifest"></html>

//cache.manifest
CACHE MANIFEST
#v0.11

CACHE:

js/app.js
css/style.css

NETWORK:
resourse/logo.png

FALLBACK:
/ /offline.html

# manifest 文件组成部分

1、CACHE 表示需要离线存储的资源列表，由包含 manifest 文件的页面将被自动离线存储，不需要把页面自身列出来

2、NETWORK 表示在它下面列出来的资源只有在在线的情况下才能访问，不会被离线存储，在离线情况下无法使用这些资源。如果 CACHE 和 NETWORK 中有一个相同的资源，那么这个资源还是会被离线存储，CACHE 优先级大于 NETWORK

3、FALLBACK 表示如果访问第一个资源失败，那么就使用第二个资源来替换，上例表示如果访问根目录下任何一个资源失败，就去访问 offline.html

# CSS

# will-change

说明：告知浏览器该元素会有哪些变化的方法，这样浏览器可以在元素属性真正发生变化之前提前做好对应的优化准备工作。 这种优化可以将一部分复杂的计算工作提前准备好，使页面的反应更为快速灵敏。

# clip-path

说明：用裁剪方式创建元素的可显示区域。区域内的部分显示，区域外的隐藏。

# filter

说明：将模糊或颜色偏移等图形效果应用于元素。滤镜通常用于调整图像、背景和边框的渲染

# 函数

# blur()

将高斯模糊应用于输入图像

# brightness()

使其看起来或多或少地变得明亮

# contrast()

可调整输入图像的对比度

# 应用

# 毛玻璃(backdrop-filter 效果或许更好)
.blur {
  width: 400px;
  height: 400px;
  border-radius: 10px;
  background: rgba(255, 255, 255, 0.5);
  position: relative;
}
.blur::before {
  content: '';
  position: absolute;
  inset: 0 0 0 0;
  background: inherit;
  filter: blur(10px);
}

# pointer-events

指定在什么情况下 (如果有) 某个特定的图形元素可以成为鼠标事件的 target
